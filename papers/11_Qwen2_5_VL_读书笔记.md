## 一、论文解决什么问题

Qwen2.5-VL 致力于构建一个**兼具强大基础能力和创新功能**的视觉-语言模型（LVLM），解决当前多模态大模型在以下方面的瓶颈：

1. **细粒度视觉感知**：现有模型在 OCR、文档解析、图表理解等方面表现参差不齐，缺乏统一的全文档解析能力。
2. **精确目标定位**：多数 LVLM 仅支持粗粒度的区域描述，缺乏边界框和点级别定位能力。
3. **长视频理解**：现有模型处理短视频尚可，但对数小时长视频的理解和秒级事件定位能力有限。
4. **Agent 能力**：在真实设备上（手机、电脑）执行复杂操作任务的能力不足。

Qwen2.5-VL 提供三个尺寸（3B、7B、72B），覆盖从边缘 AI 到高性能计算的多种场景。旗舰 72B 模型在文档理解、图表解析等方面匹敌甚至超越 GPT-4o 和 Claude 3.5 Sonnet。

## 二、核心架构设计

### 2.1 整体架构

Qwen2.5-VL 由三个组件构成：

```
图像/视频 → 重新设计的 ViT（动态分辨率）
              ↓
        MLP 视觉-语言融合器（4 patch → 1 token）
              ↓
        Qwen2.5 LLM（带 MRoPE 位置编码）
              ↓
          文本输出
```

### 2.2 重新设计的视觉编码器

| 参数 | 值 |
|------|-----|
| 隐藏维度 | 1280 |
| 层数 | 32 |
| 注意力头数 | 16 |
| 中间层维度 | 3456 |
| Patch 大小 | 14×14 |
| 窗口大小 | 112×112（8×8 patches） |
| 全注意力层索引 | {7, 15, 23, 31}（仅 4 层） |

**关键设计：窗口注意力（Window Attention）**

32 层中仅 4 层使用全自注意力，其余 28 层使用窗口注意力（窗口大小 112×112 像素）。这使得计算量与 patch 数量呈**线性关系**而非二次方关系，大幅提升推理效率。小于 112×112 的区域不做填充，保留原始分辨率。

**其他设计选择**：
- 位置编码：2D-RoPE（旋转位置编码），有效捕捉 2D 空间关系
- 视频处理：3D patch 分区，连续 2 帧为一组，显著减少输入 LLM 的 token 数
- 规范化：RMSNorm（与 LLM 保持一致）
- 激活函数：SwiGLU（与 LLM 保持一致）
- 从零训练：经历 CLIP 预训练 → 视觉-语言对齐 → 端到端微调

### 2.3 原生动态分辨率

- **空间维度**：图像按原始尺寸输入，高度和宽度调整为 28 的倍数后送入 ViT
- **不使用归一化坐标**：直接使用实际图像尺寸表示边界框和点坐标，让模型内在学习尺度信息
- 不同大小的图像映射为不同长度的 token 序列

### 2.4 动态 FPS 采样

将动态分辨率的理念扩展到时间维度：视频以不同帧率（FPS）采样，适应不同的内容变化速度。这使得模型能处理从几秒到数小时不同时长的视频。

### 2.5 绝对时间对齐的 MRoPE

Qwen2-VL 中的 MRoPE 将位置编码分解为时间、高度、宽度三个维度，但**时间 ID 绑定于输入帧数**，未考虑内容变化速度或绝对时间。

Qwen2.5-VL 的关键改进：**将时间 ID 与绝对时间（秒）对齐**。通过时间维度 ID 之间的间隔，模型能学习不同 FPS 采样率下一致的时间对齐，无需额外计算开销。这是 Qwen2.5-VL 实现秒级事件定位的核心技术。

### 2.6 MLP 视觉-语言融合器

- 将空间相邻的 4 个 patch 特征分组拼接
- 通过 2 层 MLP 投影到与 LLM 文本嵌入对齐的维度
- 相比直接用 patch 特征，显著减少 LLM 的计算量

## 三、模型规模

| 配置 | 3B | 7B | 72B |
|------|-----|-----|------|
| ViT 隐藏维度 | 1280 | 1280 | 1280 |
| ViT 层数 | 32 | 32 | 32 |
| 融合器输出维度 | 2048 | 3584 | 8192 |
| LLM 隐藏维度 | 2048 | 3584 | 8192 |
| LLM 层数 | 36 | 28 | 80 |
| LLM KV 头数 | 2 | 4 | 8 |
| 词表大小 | 151646 | 151646 | 151646 |
| 预训练 token 数 | 4.1T | 4.1T | 4.1T |

三个尺寸共享相同的 ViT 架构，仅 LLM 规模不同。

## 四、训练策略

### 4.1 预训练（三阶段）

| 阶段 | 视觉预训练 | 多模态预训练 | 长上下文预训练 |
|------|-----------|-------------|--------------|
| 数据量 | 1.5T | 2T | 0.6T |
| 序列长度 | 8192 | 8192 | 32768 |
| 训练参数 | 仅 ViT | ViT + LLM | ViT + LLM |
| 数据类型 | 图像描述、知识、OCR、纯文本 | 交错数据、VQA、视频、Agent | 长视频、长 Agent、长文档 |

**阶段 1**：仅训练 ViT，数据为图像描述、视觉知识、OCR 和纯文本，建立视觉与语言的对齐基础。

**阶段 2**：解冻所有参数，加入更复杂的数据（交错图文、VQA、数学、Agent、视频、纯文本），增强跨模态推理能力。

**阶段 3**：引入视频和 Agent 数据，序列长度扩展到 32768，提升长序列推理能力。

**训练优化**：动态打包数据样本，按 LLM 输入序列长度均匀分配计算负载。

### 4.2 预训练数据

相比 Qwen2-VL，预训练数据从 **1.2T token 扩展至 4.1T token**，包括：

- **交错图文数据**：四阶段评分过滤（文本质量、图文相关性、信息互补性、信息密度平衡）
- **绝对坐标 Grounding 数据**：使用真实图像尺寸作为坐标，涵盖 10000+ 目标类别
- **全文档解析数据**：统一为 HTML 格式（QwenVL HTML），包含表格、图表、公式、图片等
- **OCR 数据**：多语言（法、德、意、西、葡、阿、俄、日、韩、越），100 万图表样本 + 600 万表格样本
- **视频数据**：动态 FPS 采样，30 分钟以上长视频专门构造长视频描述
- **Agent 数据**：手机/网页/桌面截图 + UI 元素标注 + 多步操作轨迹 + 推理过程

### 4.3 后训练（Post-training）

分为两个阶段：

**SFT（监督微调）**：
- 约 200 万条数据（50% 纯文本 + 50% 多模态）
- ChatML 格式，支持单轮和多轮对话
- 数据过滤：两阶段管线（领域分类 → 规则 + 模型过滤）
- 拒绝采样（Rejection Sampling）：保留与 ground truth 匹配的 CoT 推理过程

**DPO（直接偏好优化）**：
- 仅使用图文和纯文本偏好数据
- 每个样本仅训练一次

后训练阶段**冻结 ViT 参数**。

## 六、关键技术贡献总结

### 6.1 窗口注意力 ViT

32 层中仅 4 层全注意力，其余 28 层窗口注意力，计算量从 O(n²) 降至 O(n)，且不影响性能。这使得动态分辨率输入变得高效可行。

### 6.2 绝对时间 MRoPE

这是实现秒级视频事件定位的核心创新。不同于 Qwen2-VL 将时间 ID 绑定于帧数，Qwen2.5-VL 将时间 ID 对齐到绝对时间（秒），使模型能：
- 在不同 FPS 采样率下保持一致的时间理解
- 精确输出秒级时间戳
- 无需额外模块（如额外检测头或文本时间戳）即可实现时间定位

### 6.3 原生坐标 Grounding

使用实际图像尺寸（而非归一化坐标）训练，模型直接感知真实尺度，在目标检测、定位、计数等任务上表现优异。

### 6.4 预训练数据从 1.2T → 4.1T

大规模高质量多模态数据是性能跃升的基础，特别是全文档解析数据（统一 HTML 格式）和 Agent 交互数据。

## 七、对本课题的启示

### 7.1 "大模型复核"模块的直接候选

Qwen2.5-VL 是课题 Stage 2.5"大模型复核"模块最直接的候选模型。课题方案计划使用 SiliconFlow API 调用 Qwen 系列 VL 模型，Qwen2.5-VL 在以下方面的能力恰好满足需求：

1. **行为验证**：给定视频片段，判断是否发生特定行为（如"人员 1 是否在喝水"）——对应视频理解能力
2. **时间定位**：精确到秒地定位行为发生的时间段——对应 Charades-STA 50.9 mIoU 的时间定位能力
3. **目标定位**：在画面中定位相关物体（如杯子、打印机）——对应 Grounding 能力

### 7.2 视频时间定位能力

Qwen2.5-VL 在 Charades-STA 上的表现（mIoU 50.9，远超 GPT-4o 的 35.7）证明了绝对时间 MRoPE 的有效性。这一能力可直接用于：
- 对 ActionFormer/BMN 输出的时间边界进行二次校验
- 基于自然语言查询定位行为时间段
- 替代或辅助传统 TAL 管线

### 7.3 与 InternVideo2 的互补

| 维度 | InternVideo2 | Qwen2.5-VL |
|------|-------------|------------|
| 定位 | 视频编码器（特征提取） | 端到端多模态 LLM |
| 优势 | 高质量视频特征 | 直接生成自然语言回答 |
| TAL | 特征 + ActionFormer | 直接时间定位 |
| 视频对话 | 通过 VideoChat2 | 原生支持 |
| 部署 | 需额外 LLM | 一体化模型 |
| API 可用性 | 有限 | SiliconFlow 等 API 可用 |

课题中可考虑：**InternVideo2 特征用于 Stage 3 TAL 管线**，**Qwen2.5-VL 用于 Stage 2.5 大模型复核**。

### 7.4 与课题方案的对接

| 论文内容 | 课题对应 |
|---------|---------|
| 视频理解 + 时间定位 | Stage 2.5 大模型行为复核 |
| 目标 Grounding | 开集目标检测（与 Grounding DINO 对比参考） |
| 长视频理解 | 监控场景长时间跨度行为分析 |
| Agent 能力 | 未来扩展：自动化标注/交互系统 |
| SiliconFlow API | 课题已规划的 VLM 调用方案 |

## 八、核心 Takeaway

1. **窗口注意力是高效 ViT 的关键**：32 层仅 4 层全注意力，计算量线性增长，使动态分辨率成为可能。
2. **绝对时间 MRoPE 实现秒级定位**：不增加任何额外模块，仅通过将时间 ID 对齐到绝对时间即可实现精确时间定位。
3. **原生坐标优于归一化坐标**：使用实际图像尺寸训练 Grounding，模型直接感知尺度信息。
4. **4.1T 预训练 token 奠定基础**：大规模高质量多模态数据是 72B 模型全面超越闭源模型的根本。
5. **多模态训练不损害纯文本能力**：甚至在代码和指令遵循上有所增强，说明视觉信息对语言理解的正面作用。
6. **三个尺寸满足不同场景**：3B 可用于边缘部署，7B 平衡性能与效率，72B 追求极致性能。
