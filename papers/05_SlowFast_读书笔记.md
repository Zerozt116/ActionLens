# SlowFast 论文读书笔记

**论文标题：** SlowFast Networks for Video Recognition

**来源：** ICCV 2019 / arXiv:1812.03982

**作者：** Christoph Feichtenhofer, Haoqi Fan, Jitendra Malik, Kaiming He (Facebook AI Research)

**论文地址：** https://arxiv.org/abs/1812.03982

**代码地址：** https://github.com/facebookresearch/SlowFast

---

## 一、论文解决什么问题

现有视频识别方法通常对空间和时间维度进行对称处理（如 3D 卷积对时空同等建模），或将运动信息编码为光流作为额外输入（双流网络）。但这些方法存在以下问题：

- 3D 卷积计算量大，且在时间维度上缺乏针对性的设计。
- 双流网络依赖手工设计的光流，非端到端学习。
- 两类方法都没有考虑到一个基本观察：**视频中的语义信息和运动信息在时间尺度上变化速度截然不同**。

论文的核心洞察来自一个简单的事实：空间语义（如"这是手"、"这是人"）变化缓慢，而运动（如拍手、挥手、行走）变化快速。因此，不需要用同样的时间分辨率来处理这两类信息。

## 二、核心设计：SlowFast 双路径架构

### 2.1 整体概念

SlowFast 网络包含两条以不同帧率运行的路径，通过侧向连接融合：

```
                    Slow 路径                     Fast 路径
                    (低帧率，强语义)               (高帧率，轻运动)
输入帧数:            T 帧 (如 4 帧)              αT 帧 (如 32 帧)
时间步长 τ:          16 (每 16 帧取 1 帧)         2 (每 2 帧取 1 帧)
通道数:              C (完整通道)                  βC (如 C/8)
时间下采样:          无 (保持 T 帧)               无 (保持 αT 帧)
计算占比:            ~80%                         ~20%
专长:                空间语义、外观、类别           运动、时间动态

        侧向连接 (Fast → Slow，每个 stage 一次)
                    ↓
            全局池化 + 拼接 + 全连接分类器
```

### 2.2 Slow 路径

- **大时间步长 τ**（典型值 16）：对 30fps 视频约每秒采样 2 帧。
- 输入 T=4 帧，对应 64 帧原始片段。
- 前几个 stage (conv1~res3) 使用**纯 2D 卷积**（时间核大小为 1），只在 res4 和 res5 才使用非退化的时间卷积。
- 原因：当时间步长大、物体移动快时，早期层的时间感受野太小，时间卷积几乎无相关性可利用；只有到后几层空间感受野够大时，时间卷积才有意义。

### 2.3 Fast 路径

- **小时间步长 τ/α**（α=8，即步长为 2）：高时间分辨率输入。
- 输入 αT=32 帧，全程**不做时间下采样**（无时间池化、无时间步长卷积）。
- **低通道容量**（β=1/8），使 Fast 路径仅占总计算量的约 20%。
- 每个 block 都使用**非退化的 3D 卷积**（时间核大小 > 1），充分建模细粒度运动。
- 参数仅 0.53M（Slow 路径为 32.4M），非常轻量。

### 2.4 侧向连接

在每个 stage（pool1、res2、res3、res4）之后将 Fast 路径特征融合到 Slow 路径。三种融合变换：

| 变换方式 | 操作 | 效果 |
|----------|------|------|
| Time-to-channel (TtoC) | 将 α 帧打包到通道维度 | 可匹配维度，但丢失时间结构 |
| Time-strided sampling | 每 α 帧取 1 帧 | 简单但丢弃大量信息 |
| Time-strided convolution | 5×1² 卷积，步长 α | **最佳**，可学习的时间下采样 |

默认使用时间步长卷积 (T-conv)，融合方式为拼接 (concatenation)。

### 2.5 生物学类比

SlowFast 的设计灵感来自灵长类视觉系统中的视网膜神经节细胞：

- **P 细胞 (~80%)**：高空间分辨率，对颜色和细节敏感，时间响应慢 → 类比 Slow 路径。
- **M 细胞 (~15-20%)**：高时间频率，对快速运动敏感，对空间细节和颜色不敏感 → 类比 Fast 路径（轻量、高帧率、弱空间能力）。

## 三、实验与结果

### 3.1 Kinetics-400 动作分类

| 模型 | Top-1 | Top-5 | GFLOPs×views |
|------|-------|-------|--------------|
| I3D (双流, ImageNet 预训练) | 75.7 | 92.0 | 216×N/A |
| Nonlocal R101 | 77.7 | 93.3 | 359×30 |
| SlowFast 4×16, R50 | 75.6 | 92.1 | 36.1×30 |
| SlowFast 8×8, R50 | 77.0 | 92.6 | 65.7×30 |
| SlowFast 8×8, R101 | 77.9 | 93.2 | 106×30 |
| SlowFast 16×8, R101+NL | 79.8 | 93.9 | 234×30 |

关键观察：SlowFast 4×16 R50 仅用 36.1 GFLOPs/view（远低于 I3D 的 216），且**无需 ImageNet 预训练**，就达到了 75.6% top-1，比之前不使用预训练的最佳结果 (73.9%) 高 1.7%。

### 3.2 AVA 动作检测

SlowFast 在 AVA v2.1 上取得了巨大提升：

| 模型 | 预训练 | Val mAP | Test mAP |
|------|--------|---------|----------|
| I3D (双流) | Kinetics-400 | 15.6 | - |
| ATR R50+NL (双流) | Kinetics-400 | 21.7 | - |
| SlowFast 8×8 R101 | Kinetics-400 | 26.3 | - |
| SlowFast 8×8 R101+NL | Kinetics-600 | 27.3 | 27.1 |
| SlowFast++ 集成 | Kinetics-600 | - | 34.3 |

SlowFast 在 AVA 上比 I3D 双流方法提升超过 **10 mAP**，且 Fast 路径的贡献达 **+5.2 mAP**（而传统光流仅带来 +1.1~1.7 mAP）。

### 3.3 核心消融实验

**Slow vs. SlowFast（Kinetics-400，R50）：**

| 模型 | Top-1 | GFLOPs |
|------|-------|--------|
| Slow-only | 72.6 | 27.3 |
| Fast-only | 51.7 | 6.4 |
| SlowFast (T-conv) | 75.6 | 36.1 |

Fast 路径单独仅 51.7%，但为 Slow 路径带来了 **+3.0%** 的提升，证明两者高度互补。

**通道容量比 β 的影响：**

| β | Top-1 | GFLOPs |
|---|-------|--------|
| 1/32 | 74.2 | 28.6 |
| 1/16 | 75.1 | 30.6 |
| 1/8 (默认) | 75.6 | 36.1 |
| 1/6 | 75.8 | 41.8 |
| 1/4 | 75.6 | 54.5 |

即使 β=1/32（Fast 路径仅增加 1.3 GFLOPs / ~5%），也能带来 **+1.6%** 提升。

**AVA 上的逐类别分析（Slow-only 19.0 mAP → SlowFast 24.2 mAP）：**

- 57/60 个类别获得提升。
- 最大绝对提升：hand clap (+27.7)、swim (+27.4)、run/jog (+18.8)、dance (+15.9)、eat (+12.5)。
- 这些类别的共同特点：**运动建模至关重要**。
- 仅 3 个类别略有下降（answer phone -0.1、lie/sleep -0.2、shoot -0.4），幅度极小。

**Fast 路径弱空间输入实验：**

| 输入 | Top-1 | GFLOPs |
|------|-------|--------|
| RGB (默认) | 75.6 | 36.1 |
| 灰度图 | 75.5 | 34.1 |
| 时间差分帧 | 74.5 | 34.2 |
| 光流 | 73.8 | 35.1 |

灰度图版本几乎与 RGB 同等效果（-0.1%），但减少约 5% FLOPs，且与 M 细胞"对颜色不敏感"的特性一致。

## 四、SlowFast 方法的关键洞见总结

1. **空间和时间不应被对称对待**：视频中语义信息变化慢、运动信息变化快，用不同帧率处理是合理且高效的。
2. **轻量级 Fast 路径的巨大价值**：仅占总计算量 20% 的 Fast 路径，在 Kinetics 上带来 +3.0%，在 AVA 上带来 +5.2 mAP。
3. **不需要光流**：SlowFast 完全从原始 RGB 端到端学习，无需手工设计的光流特征，且效果更好。
4. **两条路径高度互补**：Fast 路径单独仅 51.7%，但与 Slow 路径结合后带来显著提升，说明它学到了 Slow 路径缺失的时序动态信息。
5. **时间卷积的位置很重要**：Slow 路径在早期层使用 2D 卷积、只在后期层使用时间卷积，这是一个反直觉但有效的设计。

## 五、与本课题的关联

SlowFast 是本课题技术路线中"行为识别"模块的**核心视频动作识别骨干网络**（课题方案 §4.6 推荐）。

### 5.1 作为行为识别骨干

课题方案中提到的"SlowFast、Video Swin Transformer、MMAction2 框架"，SlowFast 是其中最成熟、工程最完善的选项之一。其优势：

- **在 AVA 上验证过**：AVA 的标注方式（以人为中心的原子动作）与本课题的任务最为接近，SlowFast 在 AVA 上的大幅领先说明它最适合本课题的场景。
- **无需光流**：端到端从 RGB 学习，部署简单，无需额外的光流计算模块。
- **预训练权重可用**：Kinetics 预训练的 SlowFast 模型可直接用于特征提取或微调。

### 5.2 与 BMN 的配合

课题方案中的"候选行为片段生成 + 行为识别"可以用 SlowFast + BMN 的组合实现：

```
视频帧 → SlowFast（提取视频特征 + 动作分类）
                ↓
        特征序列 → BMN（生成候选行为时间段 + 置信度评估）
                ↓
        候选片段 → 大模型复核（语义判断）
```

SlowFast 负责"看懂每一帧在做什么"，BMN 负责"确定行为的精确开始和结束时间"。

### 5.3 与 AVA 论文的关系

SlowFast 直接在 AVA 数据集上进行了实验，其检测架构基于 Faster R-CNN + SlowFast 骨干：

- 人员检测使用预训练的 Faster R-CNN（ResNeXt-101-FPN），AP@50 达 93.9。
- SlowFast 作为骨干提取 RoI 特征，送入 sigmoid 多标签分类器。
- 这与本课题中 AVA 论文的角色形成闭环：AVA 提供任务定义和评估标准，SlowFast 提供识别能力。

### 5.4 对课题中细粒度行为的适用性

SlowFast 在 AVA 逐类别分析中显示，对**运动密集型**行为（拍手、游泳、跑步、跳舞）提升最大。课题中的行为也类似：

- **喝水**：手部从杯子到嘴部的运动轨迹 → Fast 路径擅长。
- **打电话**：手机靠近头部的动作 → Fast 路径捕捉动态。
- **使用打印机**：人员靠近打印机、操作按钮的动作 → 需要时空结合。

### 5.5 实际部署建议

- 使用 SlowFast 8×8 R50 作为基线（65.7 GFLOPs/view，性能和效率平衡）。
- 对于长视频，可以每秒采样一次关键帧（与 AVA 1Hz 标注一致），使用 Slow 路径处理。
- Fast 路径处理每个候选片段的前后若干帧，提供运动信息。

## 六、关键 Takeaways

1. **不同的时间速度 = 不同的专长**：Slow 路径理解"是什么"，Fast 路径理解"在做什么"。
2. **轻量不等于无用**：Fast 路径仅占 20% 计算量、仅 0.53M 参数，但对运动敏感的行为类别提升巨大。
3. **端到端优于手工特征**：SlowFast 无需光流即可超越双流方法，说明从原始数据学习时空特征是更优策略。
4. **时间卷积的"何时使用"比"是否使用"更重要**：Slow 路径早期层用 2D 卷积、后期层用 3D 卷积的设计，是一个值得在课题中借鉴的实用技巧。
5. **生物学启发的架构设计**：P 细胞/M 细胞的类比说明，好的架构设计往往来自对问题本质的深入观察，而非简单的模型堆叠。
