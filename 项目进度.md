## 4. 第一阶段运行方式

查看视频元信息：

```bash
./.conda/bin/video-analyst probe path/to/video.mp4
```

运行第一阶段人员检测和跟踪：

```bash
./.conda/bin/video-analyst stage1 path/to/video.mp4 -o outputs/stage1
```

常用完整命令：

```bash
./.conda/bin/video-analyst stage1 path/to/video.mp4 \
  -o outputs/stage1 \
  --model yolo11n.pt \
  --conf 0.25 \
  --iou 0.5 \
  --imgsz 640
```

## 5. 第一阶段输出

`stage1` 命令会生成以下文件：

- `metadata.json`：视频路径、FPS、帧数、宽高、时长
- `detections.json`：逐帧人员检测框、置信度、track ID
- `tracks.json`：按人员 ID 聚合后的轨迹
- `summary.json`：本次分析摘要
- `annotated.mp4`：带人员框和 ID 的标注视频

## 5.1 第二阶段运行方式

运行第二阶段物体检测、姿态估计和规则行为识别：

```bash
./.conda/bin/video-analyst stage2 path/to/video.mp4 -o outputs/stage2
```

常用完整命令：

```bash
./.conda/bin/video-analyst stage2 path/to/video.mp4 \
  -o outputs/stage2 \
  --det-model yolo11n.pt \
  --pose-model yolo11n-pose.pt \
  --conf 0.25 \
  --iou 0.5 \
  --imgsz 640 \
  --min-event-frames 3 \
  --min-event-seconds 0.3 \
  --max-gap-frames 6
```

当前支持的规则行为：

- 喝水：`cup` 或 `bottle` 靠近人员头部和手腕。
- 打电话：`cell phone` 靠近人员头部。

第二阶段输出文件：

- `person_boxes.json`：带 track ID 的人员框。
- `objects.json`：杯子、水瓶、手机等目标物体检测结果。
- `poses.json`：匹配到人员 ID 的姿态关键点。
- `frame_action_scores.json`：逐帧行为规则命中结果。
- `events.json`：合并后的行为事件。
- `events.csv`：行为事件表格。
- `summary.json`：本次分析摘要。
- `stage2_annotated.mp4`：第二阶段标注视频。

事件合并参数：

- `--min-event-frames`：行为至少命中多少帧才输出事件。
- `--min-event-seconds`：行为持续时间至少多少秒才输出事件。
- `--max-gap-frames`：同一行为中间允许缺失多少帧仍继续合并。

第一次使用默认规则测试时，系统检测到：

- 杯子检测数量：61
- 姿态结果数量：123
- 逐帧喝水命中数量：3
- 合并行为事件数量：0

原因分析：

- YOLO 能稳定检测出 `cup`。
- ByteTrack 能稳定保持 `person_id=1`。
- YOLO Pose 能稳定输出人体姿态。
- 原喝水规则中杯子到头部距离阈值偏紧，只命中少量帧。

已调整：

- 将喝水规则从“杯子非常贴近头部”放宽为“杯子靠近头部/上半身且贴近手腕”。
- 增加 `--min-event-seconds`，过滤过短事件。
- 将默认 `--max-gap-frames` 调整为 6，允许 24FPS 视频中约 0.25 秒以内的小断点继续合并。

## 7. 当前能力边界

当前系统可以回答：

- 视频基本信息是什么
- 每一帧里检测到哪些人
- 每个人的跟踪 ID 是什么
- 每个人在视频中出现了多久
- 每个人每一帧的位置在哪里
- 能否输出标注视频供人工检查
- 视频中是否存在疑似喝水事件
- 视频中是否存在疑似打电话事件
- 行为事件的开始时间、结束时间、置信度和规则依据

当前系统还不能回答：

- 人员是否在使用打印机
- 开放类别行为查询
- 复杂多人遮挡下的高精度行为边界
- 大模型语义复核后的行为判断

这些属于后续第二阶段扩展和第三阶段内容。

## 8. 下一步建议

建议继续扩展第二阶段：行为规则基线。

推荐顺序：

1. 准备或采集包含喝水、打电话的短视频样例。
2. 基于真实样例调参：头部距离、手腕距离、最小持续帧数。
3. 增加事件关键帧截图输出。
4. 使用 Grounding DINO 支持 `printer` 等开放类别物体。
5. 实现“使用打印机”规则。
6. 接入大模型复核低置信度候选事件。

建议先做的两个行为：

- 喝水：杯子或水瓶靠近手腕和头部，并持续若干帧。
- 打电话：手机靠近头部，并持续若干帧。

暂缓的行为：

- 使用打印机：普通 YOLO 模型通常不包含打印机类别，建议后续接入 Grounding DINO 或自定义检测模型后再做。

## 10. 公开数据集调研记录

更新时间：2026-06-06

围绕本项目的目标“多人视频中谁在什么时间做了什么”，已初步检索到以下候选公开数据集：

| 数据集 | 适合用途 | 与本项目关系 | 链接 |
| --- | --- | --- | --- |
| NTU RGB+D / NTU RGB+D 120 | 喝水、打电话、玩手机等固定动作分类；包含 RGB、深度、骨架等模态 | 很适合验证喝水、打电话规则和姿态特征，但多为受控场景，不是监控视角 | https://rose1.ntu.edu.sg/dataset/actionRecognition/ |
| AVA Actions | 人级别时空动作检测，含人员框和动作标签 | 最贴近“谁在什么时间做什么”的任务定义，适合研究时空动作定位 | https://research.google.com/ava/index.html |
| Charades | 室内日常活动视频，动作和物体组合丰富 | 适合室内行为理解和人-物交互语义，但不一定直接提供精确人员 ID 跟踪 | https://prior.allenai.org/projects/charades |
| EPIC-KITCHENS | 第一视角厨房人-物交互，动作和物体标注细 | 适合学习 cup、bottle 等细粒度物体交互，但视角不是监控/第三人称 | https://epic-kitchens.github.io/ |
| MEVA | 多摄像头、多人物、真实环境活动检测 | 适合后续研究监控场景、多视角活动检测和人员重识别 | https://mevadata.org/ |
| UCF101 | 通用动作识别基准 | 可作基础模型预训练或教学验证，但不适合精确人员级时空定位 | https://www.crcv.ucf.edu/research/data-sets/ucf101/ |
| ActivityNet | 长视频时间动作定位 | 适合研究行为开始/结束时间定位，但粒度偏视频片段级 | https://activity-net.org/ |
| STAIR Actions | 100 类日常动作短视频 | 可补充日常行为样例，适合动作分类预训练或对比 | https://actions.stair.center/ |

当前优先级建议：

1. 优先看 NTU RGB+D 120：动作类别中有喝水、打电话、玩手机等，最方便验证当前第二阶段规则。
2. 同时看 AVA：更贴近人级时空动作检测，可作为论文和后续模型路线参考。
3. 如果要做真实监控/多摄像头场景，再看 MEVA。
4. 如果要强化杯子、瓶子等人-物交互，再看 EPIC-KITCHENS 或 Charades。
